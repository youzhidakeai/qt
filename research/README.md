# Research: 入场规则显性化与回测

这套工具独立于交易引擎，用于回答一个问题：**"我手动买入的直觉，能不能写成规则并被历史数据证明是正期望？"**
只有这里跑出正期望的规则，才有资格接入引擎自动下单。

## 环境

```bash
python3 -m venv .venv && .venv/bin/pip install pandas numpy
```

## 工作流

一键重跑（推荐，每隔几周积累了新交易后执行）：

```bash
./rerun.sh
```

分步等价于：

```bash
.venv/bin/python fetch_account.py        # 拉账户收益流水+成交 (需 API key, IP 加白)
.venv/bin/python build_round_trips.py    # 成交 → 回合 (0→持仓→0), BNB 手续费折算
.venv/bin/python download_klines.py      # 全市场 1m K线, 增量下载
.venv/bin/python profile_entries.py      # 入场画像: 你的买点 vs 市场随机时刻
.venv/bin/python backtest.py --v24 150 --vs 6 --r60min 3 --r5 0.5 --r15 1.0 --dh_max -3
```

## 已有结论 (2026-07-02, 数据 06/15-07/01)

- 机器人时期 (–06/21): 156 回合, 净 **-150U**。死因: 平均盈利 +3.0 < 平均亏损 -4.2, 出场结构反向。
- 手动时期 (06/22–): 200 回合, 净 **+1112U**。优势在 1-30 分钟短打 (胜率 ~80%); 拿超 30 分钟合计 -207U; 被强平 4 次。
- 手动入场画像: 只买 24h 成交额 >1 亿的热点币、1h 涨幅 +2.6%、5m 爆量 4x; 亏单大多贴着 24h 高点追。
- **入场规则机械化回测三组配置全部负期望 (盈利因子 0.55~0.73)** → 你的裁量含有 K 线之外的信息, 或者盈利有运气成分。样本翻倍后重跑再判断。

## 防自欺守则

1. **回测报告永远看"后半段 (样本外视角)"那一行**——参数是照着全时段调的，前半段好看不算数。
2. 两周数据太短，任何结论都只是"值得继续观察"，不是"可以上实弹"。
3. 调参次数越多，结果越是幻觉。每改一次参数，心里把置信度打个折。
4. 成本假设 (滑点 0.10%/边) 偏保守是故意的——实盘追拉盘的滑点只会更差。
